# Product-catalog
Тестовый проект на React. 

## Описание
Приложение демонстрирует:
- адаптивное мобильное меню
- категории товаров
- кнопки с активным состоянием
- возможность добавлять звуки при кликах

## Установка
1. Склонируйте репозиторий:
   
git clone https://github.com/Aleksei-Sabynich/Product-catalog.git
cd Product-catalog

2.Установите зависимости:
npm install


## Запуск
Запустите проект:

Локально: npm run dev
Для продакшн-сборки: npm run build


## Описание решения

В данном проекте использованы следующие технологии и подходы:  

1. **React 19**  
   - Для построения компонентного интерфейса и эффективного рендеринга.  
   - Позволяет легко создавать адаптивные и интерактивные элементы, такие как мобильное меню и кнопки с активным состоянием.  

2. **TypeScript 5**  
   - Обеспечивает статическую типизацию и предотвращает ошибки на этапе разработки.  
   - Улучшает читаемость кода и поддержку проекта.  

3. **SCSS / CSS Modules**  
   - Для стилизации компонентов с локальными классами и удобной вложенной структурой (SCSS).  
   - Обеспечивает изоляцию стилей и предотвращает конфликты.  

4. **Zustand 5**  
   - Для управления состоянием приложения (например, выбранная категория, открытое мобильное меню, корзина).  
   - Лёгкая альтернатива Redux, без лишнего boilerplate.  

5. **React Router 7**  
   - Для навигации между страницами (главная, продукт, корзина, избранное).  
   - Позволяет поддерживать SPA с красивыми URL и переходами без перезагрузки страницы.  

6. **Анимации и мобильная адаптация**  
   - Использованы CSS анимации для выпадающего меню и плавного отображения кнопок.  
   - Проект полностью адаптивен: на мобильных устройствах меню меняется на кнопку .  

7. **Деплой (опционально)**  
   - Проект можно быстро задеплоить на Netlify, Vercel или GitHub Pages через папку `dist`/`build`.  
   - Позволяет проверить работу интерфейса в продакшн-сборке.  

### Почему такой подход

- Используются современные и популярные технологии (React, TypeScript, Zustand), которые делают проект **поддерживаемым и масштабируемым**.  
- Применяется модульная структура (компоненты + SCSS Modules), что упрощает расширение функционала.  
- Адаптивность и анимации делают интерфейс более приятным для пользователя.
